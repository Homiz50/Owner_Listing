# Create Responsive Owner Property Listing Page

## Project Setup & Configuration
1.  **Install Tailwind CSS**: Install `tailwindcss`, `postcss`, and `autoprefixer`.
2.  **Initialize Tailwind**: Generate `tailwind.config.js` and `postcss.config.js`.
3.  **Configure Tailwind**: Update `tailwind.config.js` to scan source files and add Tailwind directives to `src/index.css`.
4.  **Directory Structure**: Create `src/pages` and `src/components` directories.

## Component Implementation
### 1. Reusable UI Components (`src/components/`)
*   **`Input.jsx`**: A reusable input component with label, validation support, and standard styling.
*   **`Select.jsx`**: A reusable dropdown component.
*   **`UploadButton.jsx`**: A button component handling loading states and disabled logic.

### 2. Form Logic Components (`src/components/`)
*   **`StepOneForm.jsx`**:
    *   Fields: Category, Owner Name, SubType (Dynamic), Area, Premise Name, Description, Mobile Number.
    *   Logic: Handle dynamic SubType options based on Category (Residential vs. Commercial).
    *   Validation: Ensure all fields are filled before enabling upload.
*   **`StepTwoForm.jsx`**:
    *   Fields: Full property details (rent, bhk, address, etc.).
    *   Auto-fill: Pre-populate Type, Name, and Number from Step 1 data.

### 3. Main Page & State Management (`src/pages/page.jsx`)
*   **State Management**:
    *   `stepOneData`: Store Step 1 inputs.
    *   `stepTwoData`: Store Step 2 inputs.
    *   `isStepOneUploaded`: Boolean flag for upload status.
    *   `showStepTwo`: Boolean flag for Step 2 visibility.
    *   `isLoading`: Boolean for API simulation.
*   **Flow Control**:
    *   **Upload Handler**: Simulate API call, lock Step 1, enable "Fill Other Details".
    *   **Edit Handler**: If Step 1 is modified after upload, reset `isStepOneUploaded` and hide Step 2.
    *   **Toggle Step 2**: Button to reveal Step 2 only after successful upload.
*   **Layout**: Tailwind grid layout, responsive design, step indicators.

## Integration
1.  **`src/App.jsx`**: Import and render `src/pages/page.jsx`.
2.  **`src/main.jsx`**: Ensure proper rendering.
